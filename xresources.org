** Main XTerm Configuration

   Save this as =~/.Xresources=:

   #+BEGIN_SRC xdefaults
   ! XTerm Configuration with Theme Support
   ! This file is auto-generated by theme switcher

   ! XTerm General Settings
   XTerm*faceName: Fira Mono Nerd Font
   XTerm*faceSize: 10
   XTerm*cursorBlink: true
   XTerm*saveLines: 10000
   XTerm*utf8: true
   XTerm*termName: xterm-256color
   XTerm*VT100*scrollBar: false
   XTerm*scrollTtyOutput: false
   XTerm*bellIsUrgent: false
   XTerm*title: XTerm
   XTerm*selectToClipboard: true
   XTerm*highlightSelection: true
   XTerm*trimSelection: true
   XTerm*dynamicColors: true
   XTerm*allowTitleOps: true
   XTerm*colorMode: true
   XTerm*allowWindowOps: true

   ! Keybindings
   XTerm*vt100.translations: #override \n\
     Ctrl <Key>-: smaller-vt-font() \n\
     Ctrl <Key>+: larger-vt-font() \n\
     Ctrl <Key>0: set-vt-font(d) \n\

     Ctrl Shift <Key>V: insert-selection(CLIPBOARD)
   #+END_SRC

** Theme Switcher Integration

*** Update Theme Switcher Script

    Add the following to your =~/.local/bin/theme-switcher= script:

    #+BEGIN_SRC bash
    # Apply XTerm theme
    if [ -f "$theme_dir/Xresources" ]; then
        mkdir -p ~/.config
        cp "$theme_dir/Xresources" ~/.config/Xresources
        echo "  → XTerm colors applied"
        
        # Reload Xresources if X is running
        if command -v xrdb >/dev/null && [ -n "$DISPLAY" ]; then
            xrdb -merge ~/.Xresources
            echo "  → Xresources reloaded"
        fi
    fi
    #+END_SRC

** Troubleshooting

*** Colors Not Updating
    - Ensure =xrdb -merge ~/.Xresources= is run after theme switch
    - Restart XTerm windows to see changes
    - Check if =~/.config/Xresources= exists and has correct colors

*** Font Not Loading
    - Verify Fira Mono Nerd Font is installed
    - Check font name with: =fc-list | grep "Fira Mono"=

*** Theme Not Applying
    - Verify theme files exist in =~/.config/themes/=
    - Check file permissions
    - Ensure theme switcher script is executable
